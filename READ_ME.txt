Object State Control

1. Запуск:
	- команда object_state_control --config=ИМЯ_ФАЙЛА_НАСТРОЙКИ.txt
	- простой запуск .exe файла при наличии файла config.txt в директории
   При первом запуске происходит первичная настройка программы. Нужно убедиться, что контролируемый объект находится в начальном состоянии. 

2. Настройка config.txt:
	territory_id = ter_1                                          <-   название территории  
	camera_id    = ch_1                                           <-   название камеры
	input        = rtsp://login:password@ip:port                  <-   адрес видео потока
	objects      = door_1: 19 96 393 527, door_2: 19 96 393 527   <-   название контролируемого объекта: координаты, ...

3. Координаты контролируемого объекта имеют формат: Ymin Ymax Xmin Xmax. Получить координаты объекта можно с помощью ПО LabelImg
   Пример выделения контролируемого объекта показан на скриншоте.

4. При изменении состояния объекта записывается видеофайл в директорию output/название_территории/название_камеры/название_контролируемого_объекта/время_записи.avi
   Длительность файла не более 10 сек.
   Вместе с видеофайлом сохраняется текстовый файл, в котором содержится время события. 
   В папке logs в соответствующую директорию пишутся лог файлы.

5. Если программа работает неправильно, нужно:
	- Сохранить вывод программы в консоли.
	- Найти соответствующий видеофайл.
	- Найти соответствующий лог файл (по времени).
	- Описать проблему ( например: ложные срабатывания, несрабатывания, некорректный переход в ночной режим, вылетает при наступлении определенного события и т.д.) 
   В некоторых случаях может помочь удаление файлов из директории temp (будет произведена настройка сначала).
